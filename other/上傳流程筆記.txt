# 📤 GitHub Pages 上傳流程
# ========================================

## 🔧 環境需求
- 使用 Git Bash (不要用 cmd)
- 確保已安裝 Git

## 📋 正常上傳流程

### 1 進入專案資料夾
cd "C:\Users\user\Desktop\YL-blog"

### 2 加入全部變動檔案
git add .

### 3 建立 commit 訊息
git commit -m "描述這次的更新內容"

### 4 推送到 GitHub
git push origin main

## 📋 常用 commit 訊息範例
- "更新照片故事集功能 - 全新的珍藏美好時光相簿體驗"
- "優化響應式設計和清理多餘文件"
- "更新README文檔和專案說明"
- "修復bug和改善使用者體驗"
- "新增功能: [具體功能名稱]"

# ========================================
# 🔄 問題排除：重置專案 (當遇到殘檔或衝突)
# ========================================

## ⚠️ 注意：以下操作會清空所有 Git 歷史記錄！

### 1️⃣ 刪除現有 Git 歷史
rm -rf .git

### 2️⃣ 重新初始化 Git 專案
git init

### 3️⃣ 連接遠端倉庫
git remote add origin https://github.com/symboo/YL-blog.git

### 4️⃣ 加入所有檔案
git add .

### 5️⃣ 建立初始 commit
git commit -m "重新上傳乾淨版網站"

### 6️⃣ 強制推送覆蓋遠端
git push -f origin master:main
